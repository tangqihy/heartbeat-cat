# iOS 客户端文档（Shortcuts 捷径）

iOS 是封闭平台，无法安装自定义后台 Agent。本客户端通过 **iOS 捷径（Shortcuts）自动化**实现：打开/关闭某个 App 时，自动发送心跳到服务端。

---

## 与 PC 客户端的区别

| 特性 | PC 客户端 | iOS 客户端 |
|------|-----------|-----------|
| 检测方式 | 系统 API，自动检测所有前台应用 | 手动为每个 App 配置自动化 |
| 心跳模式 | 每 10 秒连续发送 | 打开/关闭时各发送一次 |
| 服务端 Session 超时 | 30 秒 | 3 小时 |
| 覆盖范围 | 所有应用 | 仅手动配置的应用 |

---

## 系统要求

| iOS 版本 | 支持功能 |
|---------|---------|
| iOS 14 | 仅「打开时」自动化，运行前需确认 |
| iOS 15 | 仅「打开时」自动化，运行前需确认 |
| **iOS 16.4+** | 同时支持「打开时」和「关闭时」，推荐 |
| **iOS 17+** | 可完全静默运行（无通知、无弹窗），最佳体验 |

## 第一步：创建 Master Shortcut「Heartbeat」

这是一个可复用的快捷指令，所有 App 的自动化都会调用它，**只需创建一次**。

### 详细步骤

打开「快捷指令」App → 点击右上角 **`+`**（新建快捷指令）

---

#### 动作 1：接受输入

1. 点击「添加操作」
2. 搜索「接受输入」→ 选择
3. 设置：
   - 输入类型：**文本**
   - 如果没有输入：**继续**（不是「停止并响应」）

> 输入格式为 `AppName|open` 或 `AppName|close`，竖线 `|` 作为分隔符。

---

#### 动作 2：分割文本

1. 点击「添加操作」
2. 搜索「分割文本」→ 选择
3. 设置：
   - 文本：点击蓝色输入框 → 选择变量「**快捷指令输入**」
   - 在：**自定分隔符**
   - 分隔符：输入 `|`（竖线）

---

#### 动作 3：获取 App 名称（列表第一项）

1. 点击「添加操作」
2. 搜索「从列表获取」→ 选择「从列表中获取项目」
3. 设置：
   - 列表：选择变量「**分割文本**」
   - 获取：**第一项**
4. 点击「从列表获取项目」后方 → 「添加到变量」→ 新建变量，命名为 **`app_name`**

---

#### 动作 4：获取事件类型（列表最后一项）

1. 再添加一个「从列表中获取项目」
2. 设置：
   - 列表：选择变量「**分割文本**」
   - 获取：**最后一项**
3. 添加到变量，命名为 **`event`**

---

#### 动作 5：获取 URL 内容（HTTP POST）

1. 点击「添加操作」
2. 搜索「获取 URL 内容」→ 选择
3. 设置 URL：
   ```
   https://heartbeat.yourdomain.com/api/heartbeat
   ```
   （填入你的实际服务端地址）
4. 点击 URL 下方「显示更多」展开高级选项：
   - **方法**：POST
   - **标头**：点击「添加新字段」，添加两个 Header：
     - `Content-Type` → `application/json`
     - `Authorization` → `Bearer 你的API_KEY`（例如 `Bearer a1b2c3d4e5f6`）
   - **请求体**：JSON
   - **JSON 正文**：添加以下 6 个字段（点击「添加新字段」逐一添加）：

     | 键 | 类型 | 值 |
     |---|------|---|
     | `device_id` | 文本 | `iphone`（或自定义，如 `my-iphone`） |
     | `device_name` | 文本 | `iPhone`（或自定义显示名） |
     | `device_type` | 文本 | `ios` |
     | `app_name` | 变量 | 选择变量 **`app_name`** |
     | `event` | 变量 | 选择变量 **`event`** |

---

#### 命名快捷指令

点击顶部「新快捷指令」→ 改名为 **`Heartbeat`** → 点击右上角「完成」保存。

---

## 第二步：为每个 App 创建自动化

以「微信」为例，需要创建两条自动化（打开 + 关闭）。

### 创建「打开时」自动化

1. 切换到「**自动化**」Tab（底部第二个）
2. 点击右上角 **`+`** → **「个人自动化」**
3. 滚动找到 **「App」** → 点击
4. 点击「**选取**」→ 搜索并选择「**微信**」
5. 勾选「**打开时**」（取消勾选「关闭时」）→「下一步」

6. 添加第一个动作：
   - 搜索「文本」→ 选择「文本」动作
   - 输入：`微信|open`

7. 添加第二个动作：
   - 搜索「运行快捷指令」→ 选择
   - 快捷指令：选择「**Heartbeat**」
   - 点击「Heartbeat」右边的 `>` → 输入：选择第一步的**文本变量**

8. 点击「**下一步**」→ 关闭「**运行前询问**」开关 → 「**完成**」

> **iOS 17+**：关闭「运行前询问」后，自动化完全静默执行，无任何提示。
> **iOS 16**：关闭后仍会显示横幅通知，点击通知才会执行。

---

### 创建「关闭时」自动化（需要 iOS 16.4+）

步骤与上面相同，但第 5 步只勾选「**关闭时**」，文本改为 `微信|close`。

> 如果没有「关闭时」选项，说明 iOS 版本低于 16.4。此时只有「打开时」会记录，session 将在服务端 3 小时后自动超时关闭。

---

### 为更多 App 创建自动化

创建好第一个 App 的两条自动化后，可以**长按自动化 → 「拷贝」**，然后只需修改：
1. 触发器中的 App
2. 文本动作中的 App 名称

## 推荐优先追踪的 App

从这里挑 10-15 个最常用的 App 开始，后续可随时添加：

| 类别 | 常用 App |
|------|---------|
| 即时通讯 | 微信、QQ、钉钉、飞书、Telegram |
| 社交媒体 | 微博、小红书、Twitter/X |
| 短视频 | 抖音 |
| 视频 | 哔哩哔哩、YouTube、爱奇艺、腾讯视频 |
| 音乐 | 网易云音乐、QQ音乐、Spotify |
| 浏览器 | Safari、Chrome |
| 效率 | Notion、Obsidian、备忘录 |
| 购物 | 淘宝、京东 |
| 系统 | 设置、相机、照片 |

> **重要**：App 名称必须与自动化触发器中显示的**完全一致**。最安全的做法：先在触发器中选好 App，然后直接复制粘贴触发器里显示的应用名到文本动作中。

完整 App 列表见 `clients/ios/apps.json`。

## 测试验证

1. 打开「微信」
2. 等待约 2-3 秒（自动化有轻微延迟）
3. 查看服务端是否收到数据：
   ```bash
   curl "https://heartbeat.yourdomain.com/api/devices"
   # 应该看到 iphone 设备出现
   ```
4. 关闭「微信」
5. 查看 session 是否生成：
   ```bash
   YESTERDAY=$(date -d 'today 00:00' +%s 2>/dev/null || date -v0H -v0M -v0S +%s)
   TODAY_END=$(date -d 'tomorrow 00:00' +%s 2>/dev/null || date -v+1d -v0H -v0M -v0S +%s)
   curl "https://heartbeat.yourdomain.com/api/usage/timeline?start=$YESTERDAY&end=$TODAY_END&device_id=iphone"
   ```

## 常见问题

**Q: 自动化不执行（卡在「运行前询问」开关）**
A: iOS 17 之前，即使关闭了「运行前询问」，仍需要点击通知才能执行。iOS 17+ 才能真正静默。若在 iOS 16 使用，检查通知中心是否有 Heartbeat 通知等待点击。

**Q: 报错 401 Unauthorized**
A: 检查 Shortcut 动作 5 中的 `Authorization` Header：
- 格式必须是 `Bearer 你的密钥`（Bearer 后有一个空格）
- 密钥必须与服务端 `.env` 中的 `API_KEY` 完全一致（区分大小写）

**Q: 网络请求超时**
A: 在 Safari 中访问 `https://heartbeat.yourdomain.com/health`，确认手机能访问服务端。如果服务端没有 HTTPS，手机在移动数据下可能因 ATS（App Transport Security）拒绝 HTTP 请求。推荐配置 HTTPS（参见服务端文档）。

**Q: 打开 App 时有明显延迟**
A: 自动化触发本身有约 1-2 秒延迟，这是 iOS 的限制，无法避免。对数据记录精度影响很小。

**Q: 数据只有「打开」没有「关闭」**
A: 检查 iOS 版本是否 ≥ 16.4（设置 → 通用 → 关于本机）。低版本不支持「关闭时」触发器，session 会在服务端 3 小时后自动超时关闭。

**Q: 修改 Shortcut 里的服务端地址**
A: 直接编辑「Heartbeat」快捷指令中「获取 URL 内容」动作的 URL。所有自动化都调用同一个 Shortcut，只需改一处即可全部生效。

**Q: 如何批量添加 10 个 App 的自动化**
A: 目前 iOS 没有批量创建功能。创建好第一个后，利用「长按 → 拷贝」复制自动化，再逐个修改 App 名称，速度会快很多（每个约 30 秒）。
